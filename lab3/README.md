# Лабораторная работа №3: Шлюз «Умной фабрики» (Симуляция Edge/Fog)

## Цель
Смоделировать среду Edge/Fog, где данные обрабатываются локально (на "туманном" уровне) перед отправкой в Облако. Это демонстрирует работу шлюзов (Gateways) и экономию пропускной способности.

## Сценарий
На заводе установлены датчики температуры. Нужен туманный узел (шлюз), который **отфильтровывает нормальные показания** и **немедленно оповещает облако**, если температура превышает норму.

## Архитектура (всё запускается в Docker)
- **Edge Device (Датчик)**: Python-контейнер генерирует случайную температуру и публикует её в MQTT каждую секунду.
- **Fog Node (Шлюз)**: Node-RED подписывается на данные датчика.
  - Если \(T \le 80\): пишет в локальный лог (debug в Node-RED)
  - Если \(T > 80\): пересылает в облако (другой MQTT брокер)
- **Cloud (Центр)**: MQTT брокер Mosquitto + подписчик, который показывает **только критические алерты**.

### Запуск
```powershell
docker compose up --build
```

Node-RED: `http://localhost:1880`
